Pen Factory - Jeu Incrémental

Réalisé en 2025

Un jeu incrémental minimaliste à l'esthétique rétro « terminal hacker ». Fabriquez des stylos, gérez une économie dynamique avec des prix fluctuants, et découvrez des mécaniques de progression addictives. Inspiré par Universal Paperclips, ce projet explore les idle games et la gestion d'état sans framework. Le but était de ne pas utiliser d'outils IA pour coder mais uniquement pour apprendre le langages et ces spcificitées.

Voir le projet en direct Code source (GitHub)

À propos du projet

Objectif

Créer un idle game avec économie dynamique. Fabriquer des stylos, gérer l'achat de ressources à prix fluctuants, automatiser la vente et progresser vers des améliorations. Démontrer la gestion d'état complexe et les boucles de progression en JavaScript vanilla.

Technologies

JavaScript ES6 (Vanilla), HTML5, CSS3, APIs natives (DOM, setInterval)

Durée

Projet en cours (architecture modulaire, upgrades en développement)

Défis

Synchroniser plusieurs setInterval sans conflits, gérer 10+ mises à jour DOM par frame, implémenter une économie dynamique avec prix fluctuants influençant la demande, créer une progression addictive sans backend.

Snippets de code intéressants

Vente Automatique - Coeur de la Boucle de Jeu

JavaScript 
// Vente automatique des stylos toutes les secondes
function venteStylos(price) {
    while (stockStylos > 0) {
        money += price;
        stockStylos -= 1;
    }
    updateDisplay();
}

// Execution périodique
setInterval(() => venteStylos(price), 1000);

Explication

Ce pattern est le cœur des idle games. venteStylos() vend chaque stylo en stock au prix défini. La boucle while accumule les ressources, puis updateDisplay() refresh l'UI. L'objectif ? Montrer comment une action simple (vendre) répétée crée une boucle addictive.

Économie Dynamique - Réduction de Prix Aléatoires

JavaScript 
// Prix fluctuants afin de créer de la tension stratégique
function random(min, max) {
    return Math.floor(Math.random() * (max - min + 1)) + min;
}

function randomizePrices() {
    prixPlastic = 10.0 + random(0, 15);
    prixEncre = 10.0 + random(0, 15);
    updateDisplay();
    
    let delai = 5000 + Math.random() * 5000;
    setTimeout(randomizePrices, delai);
}

randomizePrices(); // Lance la boucle

Explication

Ce système génère une tension économique. Les prix changent toutes les 5-10 secondes, forçant le joueur à décider : acheter maintenant ou attendre ? La randomization crée de l'imprévisibilité. C'est une mécanique classique des tycoon/idle games.

Explication

La progression cachée crée de la satisfaction et de la découverte. À 100 stylos, l'accès aux améliorations se déverrouille. Ce pattern maintient l'engagement par surprise et objectifs clairs. Elle récompense aussi l'investissement du joueur.

Ce que j'ai appris

1

Gestion de Timers Asynchrones

Synchroniser setInterval/setTimeout sans conflits était crucial. Avoir plusieurs boucles (vente 1s, affichage 100ms, prix 5-10s) demande une architecture réfléchie. J'ai appris à compartimenter la logique pour éviter les race conditions.

2

Modularisation sans Framework

Refactoriser le code en core/ (logique), front/ (affichage), donnee/ (état) m'a montré comment structurer du JavaScript vanilla. Même sans React/Vue, une bonne séparation des responsabilités rend le code maintenable et extensible.

3

UX et Progression Addictive

L'esthétique rétro « terminal hacker » + feedback temps réel (messages au terminal) crée une immersion. Voir les chiffres augmenter, les prix baisser, les sections se déverrouiller → c'est là que naît l'envie de continuer. Le game design est aussi important que le code.
